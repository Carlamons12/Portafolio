Unidad 1: Introducción al Desarrollo Web Orientado a Servicios

1. Paradigma del desarrollo de aplicaciones

El cómputo en la nube y la orientación a servicios han cambiado la forma en que construimos software, permitiendo sistemas más flexibles y escalables.

  • Servicios en la nube: Distinción entre IaaS, PaaS y SaaS.
  • Características SOA: Reutilización, interoperabilidad y bajo acoplamiento.
  • Aplicaciones Híbridas (Mashups): Integración de datos de múltiples fuentes (APIs) para crear un nuevo servicio único.

2. Arquitectura Orientada a Servicios (SOA)

SOA es un estilo arquitectónico para construir aplicaciones de software que utilizan servicios disponibles en una red como internet.

Principios de diseño:

  • Abstracción del servicio.
  • Reutilización del servicio.
  • Autonomía del servicio.
  • Sin estado (Statelessness).

Estándares Relacionados:

  • XML (Extensible Markup Language): Estándar para el intercambio de datos estructurados.
  • SOAP (Simple Object Access Protocol): Protocolo basado en XML para intercambiar información.
  • WSDL (Web Services Description Language): Lenguaje basado en XML para describir los servicios.
  • UDDI (Universal Description, Discovery and Integration): Registro para publicar y encontrar servicios web.
  • REST (Representational State Transfer): Arquitectura ligera basada en HTTP, muy popular en la web moderna.
